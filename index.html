<article>

The <article> tag is an HTML5 element that represents an independent composition in a document, page, application, or site. This composition is intended to be independently distributable or syndicatable, meaning it should make sense on its own and could be republished elsewhere without requiring the entire surrounding document to make sense.

Usage:

Here's a simple example of how you might structure a blog page with multiple posts using the <article> tag:

        <section>
            <h1>Blog Title</h1>

            <article>
                <h2>First Blog Post Title</h2>
                <p>Introduction or summary for the first post...</p>
                <p>Main content of the post...</p>
                <!-- maybe even more content, images, etc. -->
            </article>

            <article>
                <h2>Second Blog Post Title</h2>
                <p>Introduction or summary for the second post...</p>
                <p>Main content of the post...</p>
                <!-- and so on... -->
            </article>
        </section>

<aside>

The <aside> element in HTML is used to represent content that is tangentially related to the main content around it and can be considered separate from the main content. When used within an article or web page, the <aside> content should be related to the main content, but it should also make sense to be read independently.

<figcaption>

The <figcaption> tag in HTML is used to provide a caption or a brief explanation for content within a <figure> element. The <figure> element is often used to mark up diagrams, illustrations, photos, code blocks, and other references that stand apart from the main content, and <figcaption> serves to provide context or additional information for these items.


Example

<figure>

The <figure> tag is an HTML5 element used to represent content that is related to the main flow of the document, but can be positioned apart or independently from the main content. This content, usually in the form of images, diagrams, charts, code blocks, etc., is often referenced from the main content but can be moved away without affecting the flow of the narrative.


Example

<footer>

The <footer> tag is an HTML5 element that represents the footer section of a document or a content block. It typically contains information about the author, copyright information, links to related documents, and other meta-information that doesn't fit into the flow of the main content.

<mark>

The <mark> tag is an inline element in HTML5 used to highlight parts of text, indicating those parts as being of interest or importance. Its primary purpose is to emphasize or spotlight text within a document, typically in the context of reference, search results, or to mark out specific relevance in a passage.


Example

Global warming refers to the long-term rise in Earth's average surface temperature due to human activities, most notably carbon dioxide emissions from burning fossil fuels.


<nav>

The <nav> tag is an HTML5 semantic element that represents a section of navigation links, either within the current document or to other documents. It is intended to indicate major navigation blocks on the web page, such as primary site navigation, table of contents, pagination controls, or breadcrumbs.


Example

<time>

The <time> tag is an HTML5 semantic element used to represent a specific period in time or a range of time. This could be a date, a time, or a combination of both. The element helps to make the machine-readable representation of dates and times more accessible, ensuring that the encoded information is processed consistently across web platforms and interfaces.


Example

August 12, 2023

<blockquote>

The <blockquote> tag is a semantic HTML element used to indicate that the enclosed text is a long quotation. Typically, content within a <blockquote> is cited from another source or represents a lengthy excerpt taken from another place or author.

<progress>

The <progress> tag in HTML is a semantic element used to represent the completion status of a task, such as a download, a task in a to-do list, or the loading status of a resource. This tag provides a visual indicator of the progression of a task towards its completion.

<address>

The <address> tag is a semantic HTML element used to define contact information for the author or owner of a document or an article. This element can contain various types of details, such as physical address, phone number, email address, social media links, etc. It's important to note that while the <address> tag can represent physical addresses, it's not limited to that and can be used for other forms of contact information.
